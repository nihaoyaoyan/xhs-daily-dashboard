小红书每日增长看板

这个仓库用于发布每日生成的静态 HTML 看板。看板主体包含推广分析建议、搜索词分析建议、内容分析建议，并在顶部保留高优先级运营风险提醒。

当前状态

Pages 站点骨架已上传。当前机器尚未生成真实日报文件，因为本地缺少 config/stores.json，且 Chrome 调试端口 9222 未开启。

生成并上传每日看板

  1. 在 MCP 项目中配置 config/stores.json。
  2. 启动已登录小红书商家后台的 Chrome 调试端口。
  3. 运行 uv run xhs-qianfan-daily-report --stores config/stores.json --out reports。
  4. 把生成的 reports/YYYY-MM-DD-dashboard.html 复制为本仓库的 index.html 后提交推送。
